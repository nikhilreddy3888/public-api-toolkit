The problem

AI is solving easy problems the hard way.

Browsing, scraping, and open-ended reasoning are great for messy research. They are often a poor fit for structured factual lookups that already have reliable public APIs behind them.

What happens today

  • Search for weather that already exists in an API
  • Reason about live FX rates from stale model memory
  • Scrape pages just to answer one deterministic question

What should happen instead

  • Call the relevant provider directly through MCP
  • Return structured JSON instead of noisy HTML
  • Let the model reason on top of verified current data

The product

What Public API Toolkit actually is.

Public API Toolkit is not a magical universal router. It is a practical MCP server that exposes grouped public-data tools for the kinds of questions agents should not have to browse for.

One local MCP server

Run one server and expose weather, finance, geo, dictionaries, civic data, Wikipedia, transport, media, science, and more through a clean tool surface.

41 grouped tools

Related capabilities are bundled into domain tools so agents can discover and use them without juggling a pile of narrow single-purpose installs.

Cross-client setup

Works with Codex, Claude Code, Cursor, Gemini CLI, OpenCode, and other MCP clients that support local stdio servers.

Quick install 
npx -y public-api-toolkit

Then add the same command to your MCP client config and use grouped tools like `public_api_weather`, `public_api_wikipedia`, and `public_api_currency_exchange`.

Use cases

Helpful when the answer already exists as structured data.

Public API Toolkit works best when the job is deterministic retrieval, not open-ended research. It gives agents a maintained path to live factual data without turning the whole workflow into search-and-scrape glue.

AI coding tools

Use package lookups, open data, weather, geo, UUIDs, and public facts without leaving the editor or teaching every agent a new connector.

Research assistants

Pull Wikipedia summaries, country data, open civic records, and reference information before the model synthesizes the answer.

Customer-facing AI products

Reach for structured weather, holidays, geocoding, dictionary, and transport data instead of scraping brittle websites in the middle of user flows.

MCP power users

Install one maintained server for a broad set of common public lookups instead of stitching together a dozen separate tools.
